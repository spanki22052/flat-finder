# Backend AGENTS.md

## Стек
- NestJS + TypeScript
- Postgres (локально в Docker)
- TypeORM или Prisma — выбранный ORM фиксируем здесь после старта
- JWT для аутентификации

## Структура
```
backend/src/
├── main.ts
├── app.module.ts
├── common/           # Декораторы, фильтры, перехватчики, исключения
├── config/           # Конфигурация (env, валидация)
├── database/         # Миграции, конфиг подключения
├── modules/
│   ├── auth/         # Регистрация, вход, JWT, guards
│   ├── users/        # Пользователи
│   ├── apartments/   # Квартиры, теги, статусы, парсер по ссылке (см. docs/parser.md)
│   ├── contacts/     # Контакты продавцов/арендодателей
│   ├── calls/        # Журнал звонков
│   └── reminders/    # Напоминания
└── swagger/          # Документация (опционально)
```

## Сущности ( Entities )
- User
- Apartment
- ApartmentTag / ApartmentStatus
- Contact
- Call
- Reminder

## DTO и валидация
- Для каждого модуля: create/update/list DTO.
- Валидация через class-validator, формат — единый для всех модулей.
- Никаких данных без типов на входе.

## Миграции
- Изменения схемы только через миграции ORM.
- Каждая миграция атомарна, с названием по смыслу.
- Перед миграцией — проверить, что локальная база запущена.

## API
- Версионирование: `/api/v1/`.
- Единый формат ответов: `{ data, meta?, error? }`.
- Стандартные коды ошибок и единый формат тела ошибки.

## Аутентификация
- JWT в заголовке `Authorization: Bearer ...`.
- Guards на защищённых маршрутах.
- Пароли хешируются, секреты хранятся в `.env`.

## Правила разработки
- Не смешиваем уровни ответственности внутри модулей.
- Сервис отвечает за бизнес-логику, контроллер — за роут и формат ответа.
- Логирование важных событий.
