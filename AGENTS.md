# Flat Finder — Project AGENTS.md

Этот файл описывает общие правила проекта для AI-агентов и разработчиков.

## Цель продукта
Система для сбора и работы с объявлениями о квартирах: ручной ввод, MVP-парсер по ссылке, теги/статусы, журнал звонков, простые напоминания, multi-user доступ.

## Роли и ответственность
- **Frontend**: React + Vite + FSD + Ant Design + Styled Components + Context.
  Папка: `frontend/`. AGENTS.md: `frontend/AGENTS.md`.
- **Backend**: NestJS + Postgres.
  Папка: `backend/`. AGENTS.md: `backend/AGENTS.md`.
- **Интеграция**: контракт API описан в `docs/api.md`. Изменения API синхронизируются обеими сторонами.

## Общие принципы
- В деве не используем Docker, кроме Postgres.
- Бэкенд и фронтенд разрабатываем как отдельные приложения в `backend/` и `frontend/`.
- Миграции базы — только через скрипты/инструмент бэкенда. Не правим схему вручную.
- Не коммитим секреты. Для локальной разработки `.env` в каждой части проекта.
- Используем префиксы в URL: `/api/v1/...`.
- Типизация и контракты: бэкенд публикует DTO, фронтенд генерирует/содержит типы из них.
- Изменения в API фиксируем в `docs/api.md` до реализации.

## Правила совместной работы
- Любое изменение API требует обновления `docs/api.md`.
- При добавлении поля в сущность: сначала DTO/entity, потом миграция, потом слой данных, потом UI.
- Не делаем “сокращения” в именах, если они снижают читаемость.
- Перед рефакторингом — кратко опишем текущее поведение и новое.

## Вопросы
Если требование неоднозначно — уточняем у пользователя, не импровизируем.
